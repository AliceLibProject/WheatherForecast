Создание репозитория локально
mkdir .repo.git (папка репозитория должна оканчиваться на .git)
cd .repo.git
git init --bare (создания репоизтория проекта на который можно ссылаться)
cd ../
git clone ../.ropo.git my-project-folder(клонирование мастер ветки проекта из репозитория с изменением имени)

npm init (создание package.json, задание в нем параметров)
touch .gitignore (добавить файл для того чтобы указыть какие директории не должны учитываться при изменениях в git. Туда просто нужно добавить названия скрытых от git директорий, каждую на новой строк пр. node_modules, build)

npm i -D webpack webpack-cli webpack-dev-server(webpack-cli - интерфейс командной строки, чтобы можно было задавать команды для webpack пр. npm run start ... в package.json)

npm i -D @types/react @types/react-dom (Необходимо установить эти пакеты для работы связки typescript-react)

!!! В новых версиях react корневой элемент не рендерится через ReactDOM.render, а создается через ReacDom.createRoot(...)

touch webpack.config.js

Добавить в объект module.export свойства entry, output

npm i -D html-webpack-plugin (плагин для генерации html-файла с подключенными результатами сборки.)

добавить использование плагин html-webpack-plugin (он используется для автоматического прописывания сгенерированных файлов скриптов в index.html документ с которого начинается инициализация проекта). Для этого в настройках планига при его нициализации в разделе плагинов нужно указать шаблон с которого будет создаваться итоговый файл и путь до размещения итогового файла.
!!! В шаблон можно передавать значения полей объекта options для этого плагина

npm i -D clean-webpack-plugin (для очистки директории с результатами сборки.)

добавить в раздел плагинов плагин clean-webpack-plugin, который будет очищать директопию результатов сборки от старых файлов пр. если файлы генерятся с [name][hash].bounle.js будут создаваться все новые файлы, а не переписываться старый

$ npm i -D sass sass-loader css-loader style-loader (добавляем пакеты компиляторов стилей и инициализации стилей)
sass-loader - загрузчик файлов Sass/SCSS. Позволяет понимать директиву import для стилей scss в tsx файлах
sass - компилятор файлов .scss в .css.
css-loader загрузчик CSS-файлов
style-loader загрузчик стилей в DOM. Позволяет вставлять тэг style в разметку html документа

npm install react react-dom --save (установка необходимых пакетов для реакт приложения. --save или --save-dev используются для помещения зависимых пакетов в проект или нет. --save-dev не включает зависимость в проект и используется с пакетами "остнасками" типо webpack, babel, jest и тд. Все остальные пакеты должны устанавливаться как --save)

Далее необходимо установить транспайлер, который будет использоваться webpackом для перевода tsx, jsx и других файлов кода в js. Тут можно воспользоваться 2 вариантами это npm пакеты Babel или Typescript.
При установке Babel к нему нужно подключать пресеты (это плагины реализующие различные виды транспайлинга), что потом нужно настроить в конфигурации.

npm i --save-dev @babel/core @babel/preset-env @babel/preset-react
